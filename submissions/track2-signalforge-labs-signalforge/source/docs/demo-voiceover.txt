SignalForge is a private, local-first research desk for independent investors. Public company data is abundant, but a disciplined research process is not. The product helps a user understand a business, connect fundamentals to economic and market forces, compare companies, inspect valuation scenarios, and identify what would invalidate an investment thesis. It does not predict prices, execute trades, or provide personalized investment instructions.

The architecture separates interpretation from financial authority. A typed Go control plane owns scope, permissions, evidence joins, deterministic tools, lineage, retries, review gates, and publication. A bounded specialist team uses one shared local model runtime for accounting, economics, valuation, market behavior, business strategy, evidence review, risk review, and final qualitative synthesis. Core inference runs locally on an AMD Radeon GPU through ROCm, using Gemma 4 26B A4B IT in Q4 quantization on ROCm llama.cpp. The endpoint is loopback-only, and the accepted golden journey makes no remote model calls.

This is a real live run on the allocated Radeon environment. The user asks SignalForge to compare Microsoft and NVIDIA as long-term businesses under higher-for-longer rates and slower AI infrastructure spending. The scenario controls are explicit assumptions, not forecasts. Durable case memory is optional and remains off unless the user enables it.

The Interpreter closes the intent and scope. The orchestrator creates a bounded plan and calls only the specialists required by the question. Context specialists execute concurrently through the shared local server. They exchange typed context packets and evidence references, not unrestricted chat transcripts. Two independent review roles then challenge support and risk before the final analyst produces a semantic draft.

The language model is never the calculator of record. SignalForge applies a Numerical Silence contract: models may interpret verified relationships, but Go owns every authoritative value, period, unit, formula, comparison direction, and final quantitative rendering. Twenty-eight role-authorized deterministic tools cover financial metrics, valuation, scenarios, comparisons, and scope checks. Every successful calculation produces an immutable receipt with canonical inputs, assumptions, invariants, provenance, and a replay hash. If evidence, periods, or invariants do not support a comparison, the system publishes the limitation instead of inventing precision.

The completed research case is progressive and inspectable. It exposes the comparison, transmission mechanisms, market measurements, scenarios, counterevidence, thesis-invalidation conditions, evidence, and limitations. Supported claims must survive both critics and retain their source references. System caveats are surfaced rather than hidden.

The proof layer shows exactly what crossed the publication boundary. This evidence card identifies the primary Microsoft source, the relevant section, its as-of date, citation target, and content hash. The DCF receipt was produced by the deterministic valuation engine. It exposes the formula version, verified outputs, receipt hash, source date, and invariant result. These numbers were not written by the model.

SignalForge also supports governed multi-turn research. This follow-up asks what would invalidate the thesis first. The application preserves the companies, scenario, and point-in-time boundary, while re-authorizing evidence for the new question. The response remains local, evidence-bound, and explicit about assumptions and limitations.

Durable memory is opt-in. Saved cases contain the released audit projection, not prompts, raw model responses, chain-of-thought, credentials, or private traces. The user can inspect, export, and delete a case, and every stored projection is hash-verified when read.

Radeon optimization was measured against the complete journey, not an isolated token-speed benchmark. Four context workers with unified F16 key-value cache completed the frozen workload in 157.47 seconds, 29.17 percent faster than the passing three-worker control, while both profiles passed all forty-four semantic checks. Configurations that weakened capacity or failed to improve the complete workload were rejected.

Finally, the release is designed to fail closed. Twenty-six adversarial cases exercise eleven executable gates across temporal leakage, impossible market data, prompt injection inside retrieved material, invented evidence, unauthorized tools, memory contamination, timeouts, and clean startup. All current release gates pass, but the documentation still names residual risks and the bounded Microsoft and NVIDIA scope.

SignalForge combines local specialist intelligence with deterministic financial truth. The result is not another financial chatbot. It is a private, inspectable research process built on AMD Radeon and ROCm.
