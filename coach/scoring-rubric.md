# 评分标准

## 总体评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| AI Agent 功能完整性 | 60% | 项目功能、创新性、实用性 |
| AMD GPU / ROCm 适配 | 40% | 技术实现、性能优化 |

## 详细评分标准

### 一、AI Agent 功能完整性 (60分)

#### 1. 场景定位与创意 (20分)

| 分数 | 标准 |
|------|------|
| 18-20 | 场景清晰，痛点明确，解决方案创新且实用 |
| 14-17 | 场景较清晰，解决方案合理 |
| 10-13 | 场景基本清晰，但解决方案缺乏创新 |
| 6-9 | 场景模糊，解决方案简单 |
| 0-5 | 无明确场景或解决方案 |

**验证方法**:
- 检查 README 中的问题定义
- 评估解决方案的创新性
- 考虑实际应用价值

---

#### 2. 核心能力 (20分)

| 分数 | 标准 |
|------|------|
| 18-20 | 实现 5 项核心能力，功能完整 |
| 14-17 | 实现 4 项核心能力 |
| 10-13 | 实现 3 项核心能力 |
| 6-9 | 实现 1-2 项核心能力 |
| 0-5 | 未实现核心能力 |

**核心能力清单**:
- [ ] 本地知识检索 (RAG)
- [ ] 工具调用

- [ ] 本地多轮记忆

```bash
# 多轮记忆
连续发送消息，验证上下文保持
```

### 二、AMD GPU / ROCm 适配与优化 (40分)

#### 1. GPU 推理 (20分)

| 分数 | 标准 |
|------|------|
| 18-20 | 核心推理全部在 AMD GPU 上运行 |
| 14-17 | 主要推理在 GPU 上运行 |
| 10-13 | 部分推理在 GPU 上运行 |
| 6-9 | GPU 使用率低或不稳定 |
| 0-5 | 未使用 GPU 或使用 CPU |

**验证方法**:
```bash
# 检查 GPU 使用
rocm-smi

# 检查 PyTorch GPU 支持
python -c "import torch; print(torch.cuda.is_available())"

# 检查模型是否在 GPU 上
python -c "import torch; print(next(model.parameters()).device)"
```

---

#### 2. 推理优化 (20分)

| 分数 | 标准 |
|------|------|
| 18-20 | FP16 推理 + 批处理优化，性能提升显著 |
| 14-17 | FP16 推理，有一定优化 |
| 10-13 | 基本 GPU 推理，未优化 |
| 6-9 | 推理速度慢，资源使用高 |
| 0-5 | 未进行优化 |

**验证方法**:
```bash
# GPU 显存使用
python -c "import torch; print(f'Memory: {torch.cuda.memory_allocated()/1024**3:.2f} GB')"

# 推理时间测试
python scripts/benchmark.py

# 基准对比
记录优化前后的处理时间
```

**评分参考**:
| 指标 | 优秀 (18-20) | 良好 (14-17) | 一般 (10-13) | 差 (0-9) |
|------|-------------|-------------|-------------|---------|
| 显存占用 | < 12GB | 12-16GB | 16-20GB | > 20GB |
| 单张处理时间 | < 15s | 15-30s | 30-60s | > 60s |
| 准确率 | > 98% | 95-98% | 90-95% | < 90% |
