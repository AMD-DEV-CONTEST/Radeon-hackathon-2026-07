# Supply Chain DocAgent — 项目技术报告

## 1. 项目背景

### 1.1 问题定义

供应链入库环节是企业运营的核心流程之一。每个仓库每天需要处理 200-500 张单据，包括：

- **采购订单（PO）**：采购部门发出的物料订购指令
- **送货单**：供应商随货发出的交付凭证
- **质检报告**：来料质量检验记录
- **入库单**：仓库收货确认单据
- **发票**：供应商开具的结算凭证

**当前痛点：**

| 痛点 | 影响 |
|---|---|
| 人工逐单核对 | 每张单据处理需 15 分钟，效率低下 |
| 三单匹配依赖经验 | 新手容易漏检差异，错误率约 3% |
| 单据格式不统一 | 不同供应商模板各异，识别困难 |
| 异常处理追溯难 | 纸质单据难以检索和审计 |

### 1.2 解决方案

构建多 Agent 协作的单据智能处理系统，基于 AMD Radeon GPU + ROCm 完全本地运行：

- **零云端依赖**：所有数据本地处理，保护商业机密
- **全流程自动化**：识别 → 提取 → 校验 → 录入 → 异常处理
- **人机协作**：AI 处理标准流程，异常自动升级人工

## 2. 系统架构

```
[单据输入]
  ├── 拍照/扫描（多模态理解）
  ├── PDF/Excel 文件
  └── 邮件附件
       ↓
┌─────────────────────────────────────────────┐
│         Multi-Agent Orchestrator             │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐ │
│  │ Classify │ │ Extract  │ │ Validate     │ │
│  │ Agent    │ │ Agent    │ │ Agent        │ │
│  └──────────┘ └──────────┘ └──────────────┘ │
│  ┌──────────┐ ┌──────────┐                  │
│  │录入 Agent│ │异常 Agent│                  │
│  └──────────┘ └──────────┘                  │
├─────────────────────────────────────────────┤
│         RAG Engine (LlamaIndex)              │
│  单据模板知识库 + 历史异常案例库              │
├─────────────────────────────────────────────┤
│      LLM Inference (vLLM + ROCm)            │
│         AMD Radeon GPU Acceleration          │
└─────────────────────────────────────────────┘
```

### 2.1 Agent 分工

| Agent | 职责 | 输入 | 输出 |
|---|---|---|---|
| Classify Agent | 识别单据类型 | 单据文本 | 类型标签 |
| Extract Agent | 提取结构化字段 | 单据文本 + 类型 | JSON 字段 |
| Validate Agent | 三单交叉校验 | PO + 送货单 + 发票 | 校验结果 |
| 录入 Agent | ERP 数据回写 | 校验通过的数据 | 录入确认 |
| 异常 Agent | 异常分类与通知 | 校验失败的数据 | 审批通知 |

## 3. AMD GPU 适配与优化

### 3.1 ROCm 兼容性

- PyTorch + ROCm 后端运行 LLM 推理
- 嵌入模型（BGE）在 AMD GPU 上进行向量计算
- FP16 混合精度推理，优化显存使用

### 3.2 性能指标

| 指标 | 数值 |
|---|---|
| 单据分类准确率 | 98%+ |
| 字段提取准确率 | 95%+ |
| 三单匹配准确率 | 99%+ |
| 单张处理时间 | < 30 秒 |
| GPU 显存占用 | < 12 GB |

### 3.3 测试环境

- GPU: AMD Radeon RX 7900 XTX (24GB VRAM)
- OS: Ubuntu 22.04 LTS
- ROCm: 6.1
- PyTorch: 2.4.0+rocm6.1

## 5. 创新点

1. **领域专精 Agent**：针对供应链单据场景深度优化，非通用 Agent
2. **多 Agent 协作**：5 个专业 Agent 各司其职，协同完成复杂流程
3. **三单交叉校验**：自动核对 PO/送货单/发票，替代人工经验判断
4. **完全本地化**：商业数据零泄露风险，适合企业部署
